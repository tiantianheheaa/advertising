## 一、先给结论：智能出价算法解决什么问题

**互联网广告中的智能出价**，本质是把广告主从“手动给每类流量出价”升级为“表达营销目标与约束”，由平台算法在每一次广告请求上自动决定是否参竞、出多少钱、投给谁。常见产品形态包括 **oCPC、oCPM、oCPA、eCPA、tCPA、tROAS、ROI出价、最大转化、预算约束出价** 等。2017年前后，智能出价逐步取代传统CPC，成为主流广告平台的重要投放方式。[〔1〕](https://m.blog.csdn.net/tostq/article/details/131034489)[〔5〕](https://blog.csdn.net/tostq/article/details/131034489)

从算法视角看，智能出价通常由三层组成：

1. **价值预估层**：预测这次曝光带来的点击、转化、成交、GMV、LTV等价值。
2. **出价决策层**：根据广告主目标，如CPA、ROI、预算、GMV，计算本次流量的合理出价。
3. **控制校准层**：根据实际消耗、转化成本、ROI达成情况，动态调节出价系数，避免超成本、欠消耗或ROI不达标。

# 二、智能出价算法主要有哪些

## 1. 基于eCPM换算的基础出价算法

这是智能出价的基础。广告平台最终通常要在曝光机会层面排序，因此会把不同计费方式统一换算成 **eCPM**，即千次曝光预估收益。

| 出价模式 | 典型换算逻辑 | 含义 |
|---|---|---|
| CPM | `eCPM = CPM出价` | 按曝光价值排序 |
| CPC | `eCPM = CPC出价 × pCTR × 1000` | 点击出价乘以点击率预估 |
| CPA / oCPA | `eCPM = CPA目标价 × pCTR × pCVR × 1000` | 转化目标价乘以点击率和转化率 |
| ROI / ROAS | `eCPM ≈ 预估GMV或LTV ÷ 目标ROAS × 1000` | 按预估商业价值反推出可接受广告成本 |

早期CPC广告并不是简单按CPC价格排序，而是结合预估点击率计算eCPM，例如 `eCPM = CTR × CPC × 1000`，这样平台可以在同一曝光机会下比较不同广告的预期收益。[〔7〕](https://www.opp2.com/165520.html)

**算法原理：**

- 用模型预估用户行为概率，如 **pCTR、pCVR、pPay、pLTV**；
- 将广告主目标出价换算到曝光维度；
- 在广告竞价中按eCPM、质量分、相关性、用户体验等因素综合排序。

**优点：**

- 逻辑简单，可解释性强；
- 支持不同计费方式统一竞价；
- 是oCPC、oCPA、ROI出价的基础。

**局限：**

- 强依赖pCTR、pCVR等预估模型；
- 如果转化延迟、样本少、归因不准，容易造成出价偏差；
- 对预算节奏、成本稳定性支持不足。

## 2. 固定出价与智能调价算法

在早期广告系统中，广告主通常对某类流量包、关键词、人群或广告位设置固定出价。之后平台在固定出价基础上加入 **智能调价**：广告主给一个出价上限，系统根据流量价值在上限范围内自动调高或调低出价。优质流量高出价，低质流量低出价，从而提升广告主ROI。[〔2〕](https://zhuanlan.zhihu.com/p/588970232)[〔6〕](https://zhidao.baidu.com/question/1124847907525498899.html)

**算法原理：**

可以理解为：

```text
最终出价 = 基础出价 × 流量价值系数
```

流量价值系数由以下因素决定：

- 用户点击概率；
- 用户转化概率；
- 用户历史行为；
- 广告与用户兴趣匹配度；
- 流量场景质量；
- 广告主历史转化表现。

**典型形式：**

- 智能CPC；
- 增强型CPC；
- oCPC第一阶段；
- 基于流量质量分的bid multiplier。

**优点：**

- 相比纯手动出价，能区分流量质量；
- 广告主仍保留出价上限，心理成本较低；
- 工程实现相对简单。

**局限：**

- 仍然需要广告主理解出价；
- 只能在局部流量上优化；
- 对CPA、ROI、预算消耗等全局目标控制能力有限。

---

## 3. oCPX / 目标转化成本出价算法

**oCPX** 是智能出价中最常见的一类，包括 oCPC、oCPM、oCPA 等。广告主不再只为点击或曝光出价，而是表达更深层目标，例如激活、注册、下单、付费、留资。平台算法根据转化概率自动决定每次曝光或点击的出价。[〔6〕](https://zhidao.baidu.com/question/1124847907525498899.html)[〔7〕](https://www.opp2.com/165520.html)

以目标CPA为例：

```text
单次曝光价值 ≈ 目标CPA × pCTR × pCVR
eCPM ≈ 目标CPA × pCTR × pCVR × 1000
```

如果用户点击概率高、转化概率高，系统会提高出价；如果只是容易点击但不容易转化，系统会降低出价。

**算法原理：**

1. 预测用户从曝光到点击的概率：`pCTR`
2. 预测用户从点击到转化的概率：`pCVR`
3. 根据广告主目标CPA计算本次流量价值；
4. 换算为eCPM参与竞价；
5. 后续根据真实转化成本持续校准。

**优点：**

- 广告主只需表达转化目标；
- 平台能自动识别高转化流量；
- 适合激活、注册、下单、付费、留资等效果广告。

**局限：**

- 对转化回传质量要求高；
- 冷启动阶段容易不稳定；
- 小预算、小转化量广告主的成本波动较大；
- 若广告主回传数据不完整或不真实，会影响模型优化。

## 4. 线性规划 / 约束优化出价算法

Google在2017年前后提出过将智能出价建模为 **线性规划问题** 的通用框架，用于在广告主预算、成本、ROI等约束下最大化转化量、价值或收益。其核心思想是：在全量流量机会中，选择一组最优出价，使整体目标最大，同时满足预算和成本约束。[〔1〕](https://m.blog.csdn.net/tostq/article/details/131034489)[〔5〕](https://blog.csdn.net/tostq/article/details/131034489)

典型建模方式：

```text
目标：最大化 Σ 预估价值 × 是否赢得流量
约束1：Σ 预估消耗 ≤ 广告主预算
约束2：实际CPA ≤ 目标CPA
约束3：实际ROI ≥ 目标ROI
```

通过拉格朗日乘子或对偶优化，可以得到一类形式化出价：

```text
出价 = 预估价值 × 控制参数
```

其中控制参数用于平衡预算消耗、成本达成和拿量效率。

**算法原理：**

- 把投放周期内的广告请求看成资源分配问题；
- 广告主预算是资源约束；
- 转化、GMV、ROI是优化目标；
- 求解最优流量选择和出价策略。

**优点：**

- 理论上比较优雅；
- 能同时处理预算、CPA、ROI等多个约束；
- 适合大规模、稳定流量场景。

**局限：**

- 依赖历史数据分布稳定；
- 真实竞价环境实时变化，历史求解参数可能失效；
- 大规模线上求解复杂度高；
- 通常需要叠加实时控制模块。

## 5. PID反馈控制算法

由于广告环境实时变化，单纯依赖历史数据求出来的最优出价参数往往无法稳定满足广告主目标。因此很多平台会在基础出价之上加入 **PID控制**，根据实际投放效果动态调整出价。PID的优点是简单、稳定、工程落地成熟。[〔1〕](https://m.blog.csdn.net/tostq/article/details/131034489)[〔5〕](https://blog.csdn.net/tostq/article/details/131034489)

PID包含三部分：

| 部分 | 含义 | 在广告出价中的作用 |
|---|---|---|
| P：比例项 | 当前误差 | 当前CPA高于目标，就降价；低于目标，就提价 |
| I：积分项 | 历史累计误差 | 如果长时间成本偏高，持续压低出价 |
| D：微分项 | 误差变化趋势 | 如果成本快速恶化，提前干预 |

以目标CPA为例：

```text
误差 = 实际CPA - 目标CPA
出价系数 = 原系数 - Kp×当前误差 - Ki×累计误差 - Kd×误差变化
最终出价 = 基础出价 × 出价系数
```

**算法原理：**

- 实时监控实际CPA、ROI、消耗速度；
- 计算目标与实际之间的偏差；
- 动态调节出价系数；
- 让最终成本逐步收敛到广告主目标。

**优点：**

- 可解释性强；
- 工程实现简单；
- 适合成本控制、预算节奏控制；
- 对线上稳定性友好。

**局限：**

- 原生PID不能天然保证日级CPA一定达标；
- 需要处理早晚流量差异、未来消耗预判、转化延迟；
- 小广告主转化量少，成本天然波动大，单靠出价无法完全控制。资料中也提到，小客户转化量少时，分天成本波动可能无法仅靠出价解决，有的平台会通过类似bank account的计费策略平滑成本。[〔1〕](https://m.blog.csdn.net/tostq/article/details/131034489)[〔5〕](https://blog.csdn.net/tostq/article/details/131034489)

## 6. Budget Pacing / 预算节奏控制算法

预算节奏控制是智能出价中非常关键的一类算法。它解决的问题不是“这次流量值多少钱”，而是 **预算应该在一天或一个投放周期内如何花完**。

常见策略包括：

| 策略 | 原理 |
|---|---|
| 匀速消耗 | 按时间均匀分配预算 |
| 前置消耗 | 在早高峰或优质流量时段加速消耗 |
| 后置保守 | 保留预算给晚间高转化时段 |
| 动态pacing | 根据实时竞争、转化率、预算剩余动态调节 |
| 分层pacing | 不同广告计划、商品、人群使用不同预算节奏 |

典型公式：

```text
目标消耗进度 = 当前时间进度 × 总预算
消耗偏差 = 实际消耗 - 目标消耗
出价系数 = f(消耗偏差, 剩余预算, 剩余时间, 流量质量)
```

**算法原理：**

- 如果消耗过快，降低出价或减少参竞；
- 如果消耗过慢，提高出价或扩大流量；
- 如果剩余时间短且预算多，主动提价抢量；
- 如果ROI较差，即使预算未花完，也可能降低出价。

**优点：**

- 防止预算过早花完；
- 提高全天转化稳定性；
- 是tCPA、tROAS、最大转化等出价策略的基础组件。

**局限：**

- 过度平滑可能错过优质流量；
- 激烈竞争时，预算节奏与成本目标可能冲突；
- 对转化延迟、流量峰谷预测要求高。

## 7. 拉格朗日对偶 / Primal-Dual出价算法

当广告主同时有多个目标，例如 **预算不能超、CPA不能高、ROI不能低、GMV要最大化**，出价问题就可以被建模为约束优化问题。常见解法是 **拉格朗日对偶** 或 **Primal-Dual方法**。

典型目标：

```text
最大化：转化量 / GMV / LTV
约束：消耗 ≤ 预算
约束：CPA ≤ 目标CPA
约束：ROAS ≥ 目标ROAS
```

转化为拉格朗日形式：

```text
L = 目标收益 - λ1 × 预算超额惩罚 - λ2 × CPA超标惩罚 - λ3 × ROI不达标惩罚
```

其中 `λ` 是动态更新的约束惩罚系数。

**算法原理：**

- 当CPA超目标时，提高CPA约束惩罚，系统自动降低出价；
- 当预算消耗不足时，降低预算惩罚，提高拿量倾向；
- 当ROI不达标时，提高ROI约束惩罚，减少低价值流量；
- 最终在目标收益与约束达成之间取得平衡。

资料中提到，阿里妈妈在TargetROAS计划中引入了 **Lagrange对偶梯度法** 来计算ROI约束下的出价策略梯度，并结合神经网络环境模型训练出价策略。[〔3〕](https://blog.csdn.net/alimama_tech/article/details/145021943)

**优点：**

- 适合多目标、多约束优化；
- 理论基础扎实；
- 能兼顾消耗、成本、ROI、GMV。

**局限：**

- 参数调节复杂；
- 对数据分布变化敏感；
- 需要较强的模型工程能力。

## 8. 多臂老虎机 / Bandit探索算法

Bandit算法主要解决智能出价中的 **探索与利用** 问题。广告系统不能只投已知表现好的流量，否则会错过潜在新流量；但也不能过度探索，否则会浪费预算。

典型场景：

- 不同出价系数探索；
- 不同人群包探索；
- 不同创意探索；
- 不同商品组合探索；
- 新广告主冷启动。

常见算法：

| 算法 | 原理 |
|---|---|
| ε-greedy | 大部分时间选择当前最优策略，少量时间随机探索 |
| UCB | 对不确定但可能高收益的策略给予额外探索机会 |
| Thompson Sampling | 根据后验分布采样选择策略 |
| Contextual Bandit | 结合用户、广告、场景特征做个性化探索 |

**算法原理：**

```text
策略价值 = 已知收益 + 不确定性奖励
```

如果某个流量或出价策略历史样本少，系统会给它一定探索机会；如果表现确实好，则逐渐增加投放。

**优点：**

- 适合冷启动；
- 能持续发现新增量；
- 比完全随机探索更高效。

**局限：**

- 通常只能处理短期反馈；
- 对长周期ROI、LTV、复购等目标支持不足；
- 在复杂预算约束下不如强化学习完整。

## 9. 强化学习自动出价算法

强化学习适合解决广告自动出价中的序列决策问题：一天预算如何分配？早上花多少？中午是否保守？晚上是否加速？当前出价会影响未来预算和后续竞价机会。资料中提到，从2019年开始，强化学习逐渐被用于自动出价算法。[〔3〕](https://blog.csdn.net/alimama_tech/article/details/145021943)

强化学习通常把自动出价建模为 **MDP / CMDP**：

| 元素 | 广告出价中的含义 |
|---|---|
| State 状态 | 剩余预算、剩余时间、当前消耗、转化数、CPA、ROI、竞争强度 |
| Action 动作 | 调整出价系数、提高/降低bid、选择预算节奏 |
| Reward 奖励 | 转化量、GMV、ROI、LTV、有效成交 |
| Constraint 约束 | 预算、CPA、ROI、频控、成本稳定性 |
| Policy 策略 | 在不同状态下如何调整出价 |

资料中也提到，强化学习自动出价可以建模为带约束马尔可夫决策过程，模型根据剩余时间、剩余预算、预算消耗速率等状态产出具体出价动作，奖励函数和约束函数分别表示优化目标和约束达成情况。[〔1〕](https://m.blog.csdn.net/tostq/article/details/131034489)[〔5〕](https://blog.csdn.net/tostq/article/details/131034489)

**算法原理：**

```text
当前状态 → 策略网络 → 出价动作 → 竞价环境反馈 → 更新策略
```

例如：

- 如果剩余预算多、时间少、ROI达标，则提高出价；
- 如果消耗过快、CPA超标，则降低出价；
- 如果晚高峰转化率高，则提前保留预算；
- 如果某类用户长期ROI高，则在类似流量上更激进。

**优点：**

- 适合长周期预算控制；
- 能处理序列决策；
- 可以同时优化消耗、CPA、ROI、GMV；
- 比PID更具全局视角。

**局限：**

- 训练复杂；
- 线上直接探索风险高；
- 长尾广告主样本不足时稳定性较差；
- 奖励延迟、转化延迟会影响学习效果。

---

## 10. Offline RL 离线强化学习出价算法

由于广告线上竞价环境不能随意试错，很多平台会用离线数据训练强化学习模型。资料中提到，强化学习自动出价经历了从 **SBRL：基于离线仿真器训练** 到 **ORL：直接利用线上真实数据训练** 的范式转变。[〔3〕](https://blog.csdn.net/alimama_tech/article/details/145021943)

### SBRL：Simulation-based RL

**原理：**

- 先构建一个离线竞价仿真器；
- 在仿真环境中让智能体试错；
- 学到策略后再上线。

**问题：**

- 仿真器很难还原真实竞价环境；
- 离线效果和线上效果可能存在较大gap。

### ORL：Offline RL

**原理：**

- 不再依赖人工仿真器；
- 直接使用线上历史真实投放数据训练策略；
- 避免线上试错风险。

**问题：**

- 历史数据通常由旧模型产生；
- 状态-动作空间覆盖有限；
- 新策略容易被限制在历史策略附近，优化幅度有限。资料中也指出，ORL受线上真实数据集覆盖空间有限影响，策略通常被限制在数据集附近，优化幅度较小。[〔3〕](https://blog.csdn.net/alimama_tech/article/details/145021943)

---

## 11. Model-based RL / MBRL自动出价算法

MBRL是近年来较新的强化学习出价范式。资料中提到，MBRL的基本思路是：基于线上真实数据训练一个神经网络环境模型，用它模拟线上竞价环境，再生成经过置信度处理的离线虚拟数据，与真实数据一起训练出价策略，形成“虚实融合”的训练方式。[〔3〕](https://blog.csdn.net/alimama_tech/article/details/145021943)

**算法原理：**

```text
线上真实数据
      ↓
训练神经网络环境模型
      ↓
模拟不同出价动作下的消耗、转化、ROI变化
      ↓
生成虚拟训练样本
      ↓
与真实数据混合训练强化学习策略
      ↓
输出自动出价策略
```

环境模型一般输入：

- 当前状态；
- 历史消耗；
- 剩余预算；
- 剩余时间；
- 当前出价动作；
- 历史ROI、CPA、GMV等。

输出：

- 下一状态；
- 预估消耗；
- 预估转化；
- 预估奖励；
- 结果不确定性。

资料中提到，MBRL通过神经网络环境模型缓解SBRL仿真器不准确问题，同时通过虚拟数据扩充缓解ORL数据覆盖不足问题；在阿里妈妈万相台无界版多个场景中完成扩流，并带来消耗和GMV收益提升。[〔3〕](https://blog.csdn.net/alimama_tech/article/details/145021943)

**优点：**

- 比传统仿真器更接近真实环境；
- 比纯Offline RL拥有更强探索能力；
- 适合预算约束、ROI约束等复杂场景；
- 可通过置信度处理降低模型错误带来的风险。

**局限：**

- 环境模型训练难度高；
- 如果环境模型错误，可能误导策略；
- 需要复杂的离线评估和线上灰度机制。

---

## 12. 深度拍卖 / Neural Auction相关算法

严格来说，Deep Auction、DeepGSP、Neural Auction更偏向 **拍卖机制和排序计费机制优化**，不完全等同于广告主侧智能出价，但它们会影响智能出价的最终竞价结果。

资料中提到，Google曾提出通过深度神经网络完成广告拍卖和计费，希望在平台收益和激励兼容之间取得平衡；阿里也提出过DeepGSP、Neural Auction等方法，用深度学习或强化学习优化分配概率、rank score和多目标收益。[〔1〕](https://m.blog.csdn.net/tostq/article/details/131034489)[〔5〕](https://blog.csdn.net/tostq/article/details/131034489)

**算法原理：**

- 用神经网络学习广告排序和计费规则；
- 将bid、pCTR、pCVR、GMV、用户体验等特征输入模型；
- 输出rank score、胜出概率或计费参数；
- 通过单调性约束、regret约束、min-max网络等方式保证机制合理性。

**优点：**

- 能处理复杂多目标；
- 可同时优化平台收入、广告主价值、GMV、用户体验；
- 在多广告位、多商品、多目标场景下更灵活。

**局限：**

- 可解释性较弱；
- 机制设计复杂；
- 需要保证激励兼容、单调性和计费可反解。

# 三、按产品目标划分：常见智能出价策略

| 策略 | 广告主表达 | 算法核心 | 适用场景 |
|---|---|---|---|
| 最大点击 | 在预算内拿更多点击 | pCTR预估 + CPC控制 | 流量引入、内容浏览 |
| 最大转化 | 在预算内拿更多转化 | pCVR预估 + 预算pacing | 激活、注册、留资 |
| 目标CPA / tCPA | 每个转化成本控制在目标附近 | CPA约束优化 + PID/LP/RL | 效果广告主主流模式 |
| oCPC | 按点击计费，但优化转化 | pCTR×pCVR换算 + 成本控制 | 从CPC升级到转化优化 |
| oCPM | 按曝光计费，优化转化 | 曝光层直接计算转化价值 | 信息流、短视频常见 |
| 目标ROAS / tROAS | 保证广告投入产出比 | 预估GMV/LTV + ROI约束 | 电商、游戏、本地生活 |
| 预算约束出价 BCB | 花完预算并最大化价值 | 预算pacing + RL/LP | 大预算广告主 |
| 价值最大化 | 最大化GMV、LTV或利润 | pLTV/pGMV预估 + 约束优化 | 电商、金融、游戏 |
| 自动放量 | 在成本可控下扩大规模 | 探索算法 + 相似流量扩展 | 成熟计划扩量 |

# 四、智能出价算法的发展历程

## 阶段1：固定出价阶段

早期互联网广告以 **CPM、CPC固定出价** 为主。广告主对流量包、关键词、人群或广告位设置固定价格，系统按出价或换算后的eCPM排序。CPM模式下可理解为 `bid_m / 1000`，CPC模式下则结合pCTR计算流量价值。[〔2〕](https://zhuanlan.zhihu.com/p/588970232)[〔6〕](https://zhidao.baidu.com/question/1124847907525498899.html)

**特点：**

- 广告主手动调价；
- 平台主要做排序和计费；
- 优化依赖投放人员经验；
- 难以精细区分用户价值。

---

## 阶段2：eCPM排序与质量预估阶段

随着广告平台数据积累，系统开始引入 **pCTR、质量分、相关性、落地页质量** 等模型。广告不再简单按出价排序，而是结合点击率、转化率、广告质量进行综合排序。CPC广告中，平台通常通过 `eCPM = CTR × CPC × 1000` 把点击出价换算为曝光价值。[〔7〕](https://www.opp2.com/165520.html)

**特点：**

- 从“谁出价高谁赢”转向“谁预估价值高谁赢”；
- 机器学习开始参与广告排序；
- 广告质量和用户体验被纳入系统。

---

## 阶段3：智能调价阶段

在固定出价基础上，平台开始根据流量价值自动调节出价。广告主设置出价上限，系统在上限范围内对不同用户、不同场景、不同流量质量进行差异化出价。资料中提到，这一阶段的产品形态开始以OCPX出现，把价值锚定到可预估指标上。[〔2〕](https://zhuanlan.zhihu.com/p/588970232)[〔6〕](https://zhidao.baidu.com/question/1124847907525498899.html)

**特点：**

- 低质流量低出价；
- 高质流量高出价；
- 广告主仍需设置基础出价；
- 算法主要做局部流量价值修正。

---

## 阶段4：oCPX / 自动出价阶段

2016—2017年前后，行业开始大规模进入智能出价阶段。广告主不再只表达点击价格，而是表达预算、CPA、ROI等营销目标，平台自动在流量维度出价。资料中提到，2017年前后，智能出价产品，如自动出价、oCPC、eCPA、CPA、ROI等，逐步取代CPC成为主流。[〔1〕](https://m.blog.csdn.net/tostq/article/details/131034489)[〔5〕](https://blog.csdn.net/tostq/article/details/131034489)

**特点：**

- 广告主从“买点击”变为“买转化”；
- 平台从“排序系统”升级为“自动决策系统”；
- 出价与计费逐步解耦；
- 转化回传和模型预估变得非常关键。

---

## 阶段5：约束优化与PID控制阶段

当oCPX大规模上线后，平台面临核心问题：如何既帮助广告主拿量，又稳定满足CPA、ROI、预算等约束。于是线性规划、拉格朗日优化、PID控制、预算pacing等方法被广泛应用。资料中提到，Google曾将智能出价建模为线性规划问题，而实际工程中通常还需要额外控制模块，如PID控制，以实时满足客户约束。[〔1〕](https://m.blog.csdn.net/tostq/article/details/131034489)[〔5〕](https://blog.csdn.net/tostq/article/details/131034489)

**特点：**

- 从单次流量价值判断，升级为周期级目标控制；
- 关注日级CPA、ROI和预算达成；
- PID因简单稳定被广泛使用；
- 但小样本、转化延迟、流量波动仍是难点。

---

## 阶段6：强化学习自动出价阶段

从2019年前后开始，强化学习逐渐用于自动出价。原因是广告出价天然是序列决策问题：当前消耗会影响未来预算，当前提价会影响后续拿量，当前保守可能错过优质流量。资料中提到，强化学习自动出价算法从2019年开始逐渐应用，并成为智能投放领域的重要技术。[〔3〕](https://blog.csdn.net/alimama_tech/article/details/145021943)

**特点：**

- 将广告投放建模为MDP或CMDP；
- 状态包括剩余预算、剩余时间、消耗速率等；
- 动作是调整出价；
- 奖励是转化、GMV、ROI等；
- 约束是预算、CPA、ROI达成。

---

## 阶段7：SBRL → ORL → MBRL阶段

强化学习出价的训练范式也在演进。资料中提到，早期多采用 **SBRL**，即基于离线仿真器训练；之后转向 **ORL**，即直接利用线上真实数据训练；再进一步发展到 **MBRL**，用神经网络环境模型生成可信虚拟数据，与真实数据混合训练。[〔3〕](https://blog.csdn.net/alimama_tech/article/details/145021943)

| 阶段 | 方法 | 核心问题 | 改进方向 |
|---|---|---|---|
| SBRL | 离线仿真器训练 | 仿真器与线上环境gap大 | 提高仿真真实性 |
| ORL | 线上真实历史数据训练 | 数据覆盖空间有限 | 增强探索能力 |
| MBRL | 神经网络环境模型 + 虚实融合训练 | 环境模型误差风险 | 置信度处理、稳定训练 |

资料中提到，MBRL通过深度学习环境模型提取线上竞价环境信息，缓解SBRL的离线不一致问题，同时通过可靠虚拟数据扩充训练集，突破ORL只能在历史数据附近优化的限制。[〔3〕](https://blog.csdn.net/alimama_tech/article/details/145021943)

---

## 阶段8：多目标、深度机制与全链路智能阶段

当前智能出价正在从单一CPA优化，走向 **多目标联合优化**，例如同时优化消耗、GMV、ROI、广告主长期价值、平台收入和用户体验。DeepGSP、Neural Auction、TargetROAS、MBRL等方向都体现了这一趋势。资料中提到，DeepGSP不仅关注平台收益，还包含GMV等多目标优化；Neural Auction则进一步把bid融入rank score计算模型。[〔5〕](https://blog.csdn.net/tostq/article/details/131034489)

**特点：**

- 从单计划优化走向全局优化；
- 从短期转化走向长期LTV；
- 从单目标CPA走向ROI、GMV、利润、用户体验多目标；
- 从手工规则走向深度学习和强化学习结合。

---

# 五、智能出价算法的典型系统架构

一个成熟广告平台的智能出价系统通常如下：

```text
用户请求
  ↓
用户画像 / 场景特征 / 广告位特征
  ↓
候选广告召回
  ↓
pCTR / pCVR / pGMV / pLTV预估
  ↓
目标出价模块
  - oCPA
  - tCPA
  - tROAS
  - 最大转化
  - 预算约束出价
  ↓
控制模块
  - PID
  - pacing
  - 拉格朗日约束
  - RL策略
  ↓
eCPM排序
  ↓
拍卖计费
  ↓
广告展示
  ↓
点击 / 转化 / GMV / ROI回传
  ↓
模型训练与策略更新
```

其中，**价值预估模型决定流量值不值钱，控制算法决定当前该不该激进，拍卖机制决定最终能否赢得展示。**

# 六、不同算法的适用场景对比

| 算法 | 适用阶段 | 适合广告主 | 主要优势 | 主要风险 |
|---|---|---|---|---|
| eCPM换算 | 所有阶段 | 全部 | 简单、统一竞价 | 依赖预估准确性 |
| 智能调价 | 初级智能投放 | 中小广告主 | 易理解、低风险 | 全局控制弱 |
| oCPX | 效果广告 | 转化目标明确广告主 | 直接优化转化 | 冷启动、回传质量 |
| 线性规划 | 大规模稳定投放 | 大预算广告主 | 理论最优 | 对环境变化敏感 |
| PID控制 | 成本稳定控制 | 大多数广告主 | 稳定、可解释 | 处理长期决策弱 |
| Budget Pacing | 预算管理 | 预算敏感广告主 | 消耗稳定 | 可能错过高峰流量 |
| 拉格朗日优化 | 多约束投放 | ROI/CPA要求严格广告主 | 多目标平衡 | 参数复杂 |
| Bandit | 冷启动、探索 | 新计划、新商品 | 探索效率高 | 长期目标弱 |
| RL | 长周期自动出价 | 大预算、复杂目标广告主 | 序列决策能力强 | 训练和上线复杂 |
| MBRL | 高阶自动出价 | 大平台、大广告主 | 兼顾探索与稳定 | 环境模型难训练 |
| Neural Auction | 平台机制优化 | 平台侧 | 多目标排序计费 | 可解释性与机制约束难 |

# 七、关键挑战

## 1. 转化延迟

很多转化不是点击后立即发生，例如下单、付费、复购、LTV。这会导致系统在短时间内无法准确判断某次出价是否有效。

## 2. 冷启动

新广告主、新商品、新计划缺少历史数据，pCVR和pLTV预估不稳定。通常需要Bandit探索、相似广告迁移、行业先验等方法。

## 3. 小样本波动

小广告主转化量少，即使算法合理，日级CPA也可能大幅波动。资料中也提到，小客户转化量较少时，其分天实际转化成本存在天然波动，这种波动并非完全能通过出价控制。[〔1〕](https://m.blog.csdn.net/tostq/article/details/131034489)[〔5〕](https://blog.csdn.net/tostq/article/details/131034489)

## 4. 目标冲突

广告主希望 **成本低、量大、ROI高**，但这三者经常冲突。智能出价的核心不是让所有指标同时最优，而是在约束下做最优权衡。

## 5. 数据质量

智能出价依赖转化回传、商品数据、用户行为数据。如果回传延迟、漏传、错传或归因规则变化，模型会出现偏差。

---

# 八、总结

**智能出价算法的发展主线** 可以概括为：

```text
固定出价
→ eCPM换算
→ 智能调价
→ oCPX自动出价
→ 线性规划 / 约束优化
→ PID + pacing实时控制
→ 强化学习自动出价
→ Offline RL / MBRL
→ 多目标深度拍卖与全链路智能优化
```

**核心结论：**

1. **eCPM换算** 是所有广告竞价的基础。
2. **oCPX** 是智能出价的主流产品形态。
3. **PID和pacing** 是工程上最常用、最稳定的控制算法。
4. **线性规划和拉格朗日优化** 适合处理预算、CPA、ROI等约束。
5. **强化学习和MBRL** 是面向复杂序列决策的高级自动出价方向。
6. 智能出价最终竞争力取决于三件事：**预估准、控制稳、反馈快**。
